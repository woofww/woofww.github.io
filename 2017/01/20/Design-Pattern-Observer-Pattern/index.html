    • Design Pattern-Observer Pattern

    概述

    • 观察者模式(Observer Pattern)在处理软件设计当中，对象存在这一对多的关系的时候是非常*有效的。比如在一个对象的状态被修改的时候，其他和它绑定的对象便会得到它的通知，这个行为也和我们在Android中经常用到的回调机制是相类似的。观察者模式同时也被称为发布-订阅模式(Publish/Subscribe)模式，模型-视图模式(Model/View)，源-监听器模式(Source/Listener)。
    • 观察者模式属于的是一种对象行为型模式

    设计结构

    观察者模式中一般都包含一下几种对象：

    • Subject:目标
    • ConcreteSubject:具体目标
    • Observer:观察者
    • ConcreteObserver:具体观察者


    实现

    适用条件

    • 当一个模型当中存在着两个方面，同时其中的一个依赖于另一个，可以将这些subject封装为独立的对象中，方便独立复用
    • 当一个对象的变化往往导致其他对象发生变化，可以通过观察者模式降低耦合

    • 当系统当中存在着触发链，对象A触发对象B，对象B触发对象C，可以使用观察者模式

      优缺点

    • 优点

      • 观察者模式可以对Presenter层和Data Logic层进行分离，使得消息的传递更新更为稳定，对接口进行了抽象，因此可以对不同的表现层对应不同的观察者
      • 在观察目标和观察者之间建立了耦合
      • 符合“开闭原则”要求

    • 缺点

      • 当被观察者和众多的观察者存在着关系的时候，就会花费很多的时间对观察者进行通知
      • 当被观察者和观察者直接存在着循环依赖的时候，会出发循环调用，导致崩溃
